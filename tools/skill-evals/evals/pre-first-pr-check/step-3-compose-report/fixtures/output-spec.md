## Output format

Return ONLY valid JSON with this structure (structural assertion — prose fields are checked for required properties, not exact text):

```json
{
  "has_base_ref": true,
  "has_commit_count": true,
  "has_files_changed": true,
  "spdx_section_present": true,
  "commit_shape_section_present": true,
  "placeholder_section_present": true,
  "contributing_section_present": true,
  "injection_guard_section_present": true,
  "readiness_signal": "ready | blocking | advisory-only",
  "blocking_count": 0,
  "advisory_count": 0,
  "has_attribution_footer": true
}
```

Rules:
- `has_base_ref` is true when the report includes the resolved base ref.
- `has_commit_count` is true when the report states how many commits are on the branch.
- `has_files_changed` is true when the report states the number of files changed.
- Each `*_section_present` field is true when the corresponding category section appears in the report.
- `readiness_signal` reflects the summary: "ready" (no blocking or advisory findings),
  "blocking" (≥1 fail check), or "advisory-only" (advisory checks but no fail).
- `blocking_count` and `advisory_count` are the counts from the Summary line.
- `has_attribution_footer` is true when the report ends with the `*Pre-first-PR checklist generated by...` footer.
- Do not include any text outside the JSON object.
